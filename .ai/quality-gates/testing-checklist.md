# Quality Gate: Testing

- [ ] Existing tests pass.
- [ ] Important new behaviour has suitable coverage (see [testing skill](../skills/testing.md) priorities).
- [ ] The core journey was manually exercised in a running app.
- [ ] Form success **and** error states were checked (where forms are involved).
- [ ] Responsive behaviour was checked at 375 / 768 / 1024 / 1440px.
- [ ] Existing failures are clearly distinguished from new failures.
- [ ] Build warnings were reviewed, not ignored.
- [ ] Visual changes were checked in a preview (screenshots in `.ai/artifacts/current/` where practical).
